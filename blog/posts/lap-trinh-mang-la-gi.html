Lập trình mạng là gì? Kiến thức cần biết cho sinh viên

Lập trình mạng là viết chương trình có thể gửi/nhận dữ liệu qua mạng. Ví dụ: chat app, đăng nhập tài khoản, gọi API, game online, hoặc web app (frontend gọi backend).

1) Mô hình Client – Server

  • Client: phía người dùng (trình duyệt, app mobile).
  • Server: nơi xử lý nghiệp vụ, trả dữ liệu (API, database).

Khi bạn mở web, trình duyệt gửi request tới server, server trả response. Đó là lập trình mạng theo kiểu web.

2) TCP/UDP (hiểu nhanh)

  • TCP: ưu tiên đúng dữ liệu, đủ gói, đúng thứ tự (ổn định hơn).
  • UDP: ưu tiên tốc độ, có thể mất gói (hay dùng trong game, streaming).

3) HTTP/HTTPS (thứ bạn gặp hằng ngày)

HTTP là giao thức dùng cho web. Khi bạn gọi API bằng JavaScript (fetch) hoặc backend Java gọi API, thường là HTTP.

  • GET: lấy dữ liệu
  • POST: gửi dữ liệu (tạo mới)
  • PUT/PATCH: cập nhật
  • DELETE: xoá

4) Socket là gì?

Socket là “đầu nối” để 2 chương trình nói chuyện qua mạng. Bạn sẽ gặp khi làm chat realtime, game, hoặc bài tập TCP client/server.

5) Học theo lộ trình nào?

  • Bắt đầu với HTTP + REST API (dễ áp dụng cho web/đồ án).
  • Sau đó học Socket TCP để hiểu nền tảng.
  • Cuối cùng tìm hiểu realtime (WebSocket) nếu cần.
